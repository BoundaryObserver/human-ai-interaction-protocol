# 人机交互协议理论

### 一个观察者的系统性归纳

---

## 这是什么

一套基于与大语言模型（LLM）独立交互观察而归纳出的交互校准理论体系。

核心主张：当前主流大语言模型存在可识别的系统性行为倾向（路径依赖），这些倾向可以通过外部校准指令加以约束，但无法通过指令完全消除——因为它们源于模型的训练目标本身，而非指令的不足。

## 核心内容

- **八条定律**：从"目标替代"到"策略去情境化律"，构成完整的因果链
- **路径依赖清单（30条）**：AI交互中的系统性行为倾向，按A–I九类结构化分类
- **终极协议**：理论体系的实践封装——一组可直接使用的校准指令
- **交互方法论**：坦诚→协议→校正的协作循环

## 这不是什么

- 不是人机交互（HCI）领域的界面设计研究
- 不是基于实验室实验的实证论文
- 不是对模型内部机制的训练层面分析
- 不适用于非Transformer架构的AI系统

## 适用范围

本体系适用于当前主流基于Transformer架构的大语言模型。理论基于外部交互观察，不涉及模型内部参数或训练过程。

## 谁应该读

- AI / Prompt 工程师
- 依赖AI协作的内容创作者
- 关注AI行为规律的研究者

对AI完全不了解的读者可能难以理解核心概念。

## 创作者声明

本体系的核心概念、定律和路径依赖清单均由作者（边界观察者）在与多个大语言模型（ChatGPT、DeepSeek、Claude）的独立交互中归纳提出，未引用任何已有学术论文或理论框架。

文档的文字撰写过程中使用了AI辅助润色，但所有理论判断、概念定义和校准指令均由作者本人完成并交叉验证。
